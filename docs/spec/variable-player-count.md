# 스펙: 가변 참가 인원 (2~6명)

> 상태: **[구현완료]**

## 배경
기존 게임은 참가자가 **3명 고정**(`index.html`의 `const N = 3`)이었다. 사용자가 인원을 자유롭게 조절할 수 있게 한다.

## 동작

### 입력 → 결과
- **셋업 화면 진입 시**: 기본 3명의 참가자 행이 표시된다 (기존과 동일).
- **`＋ 참가자 추가` 버튼 클릭**: 참가자 행이 1개 추가된다. 새 참가자는 빈 이름 + 기존에 쓰이지 않은 색을 부여받는다.
- **참가자 행의 `✕` 버튼 클릭**: 해당 행이 삭제되고, 아래 행들이 위로 당겨진다. 각 행의 입력 이름/색은 데이터와 함께 이동한다.
- **출발 시**: 현재 참가자 수(N)만큼 레인이 생성되고 경주가 진행되며, 결과 순위도 N명 전체를 표시한다.

### 조건/제약
- **최소 2명, 최대 6명** (`MIN_N = 2`, `MAX_N = 6`).
- 참가자 수가 2명이면 모든 `✕`(삭제) 버튼이 숨겨진다 (2명 미만 불가).
- 참가자 수가 6명이면 `＋ 참가자 추가` 버튼이 숨겨진다 (6명 초과 불가).
- 색상은 기존 `PALETTE`(7색)에서 부여하며, 최대 6명이므로 **전원이 서로 다른 색**을 가진다.
- 꼴찌 지정(rig) 대상자가 삭제되면 지정이 무효화되어 자동으로 랜덤 꼴찌가 된다.

### 엣지 케이스
- 2명에서 ✕ 시도 → 버튼이 없으므로 불가 (방어).
- 6명에서 추가 시도 → 버튼이 없으므로 불가 (방어).
- 중간 행 삭제 → `[A,B,C]`에서 B 삭제 시 `[A,C]`가 되고 C가 2번 행으로 이동, 이름/색 보존.
- 결과 화면 메달: 5위까지 이모지(🥇🥈🥉4️⃣5️⃣), 6위는 6️⃣로 표시.

## 영향 범위
- `index.html`
  - 스크립트: `N`을 `const`→`let`, `MIN_N`/`MAX_N` 추가, `renderSetup()`에 추가/삭제 버튼 렌더링, `addRacer()`/`removeRacer()`/`nextHue()` 추가, `medals` 배열에 6️⃣ 추가.
  - 스타일: `.del-btn`, `.add-btn` 추가 및 가로 모드 미디어쿼리 대응.
- `sw.js`: `CACHE` 버전 `v9` → `v10` (변경 반영).
- 신규 테스트: `tests/player-count.spec.js`, `playwright.config.js`, `package.json`.

## 완료 기준
`npx playwright test`가 전체 통과한다. 테스트는 다음을 검증한다:
1. 기본 진입 시 참가자 행 3개.
2. 추가 버튼으로 6명까지 늘어나고, 6명에서 추가 버튼이 사라진다.
3. 2명까지 줄이면 삭제 버튼이 사라진다.
4. N명으로 출발하면 레인 N개가 생성되고 결과 순위가 N개 표시된다.

**검증 결과 (2026-06-20):** `npx playwright test` — 6/6 통과 (chromium). 구현이 스펙과 일치함(Gate 6 대조 완료).
